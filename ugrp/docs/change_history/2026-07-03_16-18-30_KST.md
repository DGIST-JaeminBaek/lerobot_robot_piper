## 시작 자세 틀어짐 원인

- 기존 follower 연결 흐름은 `connect(calibrate=True)`에서 항상 `parking()` 실행
- 전원 재투입 후 손으로 수평 정렬한 첫 실행은 물리 자세와 코드 초기 자세가 우연히 맞음
- 프로그램만 재실행하면 follower가 다시 `INITIALIZE_POSITION`으로 이동
- leader는 현재 자세 또는 이전 제어 상태를 기준으로 action 생성
- 결과적으로 leader/follower 시작 기준이 달라져 재실행 직후 틀어짐 발생

## 적용된 보정 구조

- `PARK_ON_CONNECT=false`
  - teleop 시작 시 follower 강제 parking 비활성화
  - 프로그램 재실행 시 현재 follower 자세 유지
- `USE_ACTION_OFFSET=true`
  - 첫 leader action과 follower 현재 joint position 차이 자동 계산
  - 이후 follower goal에 offset을 더해 상대 추종
- `USE_MANUAL_ACTION_OFFSET=true`
  - 자동 계산 대신 `ACTION_OFFSET_*` 값을 고정 사용
  - offset report를 보고 반복 실험용 값으로 고정할 때 사용

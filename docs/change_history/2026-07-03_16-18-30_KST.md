## 1. LeRobot 0.4 호환성 보완

WEGO 원본은 `MotorsBusBase` 기반 구현입니다. 로컬 수정본은 LeRobot 0.4 계열의 `MotorsBus` 추상 인터페이스에 맞춰 `PiperMotorsBus`를 수정했습니다.

Before: WEGO 원본 `lerobot_robot_piper/motors/piper_motors_bus.py:8`

```python
from lerobot.motors.motors_bus import MotorsBusBase

class PiperMotorsBus(MotorsBusBase):
    apply_drive_mode = False
```

After: 현재 `lerobot_robot_piper/motors/piper_motors_bus.py:8`

```python
from lerobot.motors.motors_bus import MotorsBus

class PiperMotorsBus(MotorsBus):
    apply_drive_mode = False
    available_baudrates = [1000000]
    default_baudrate = 1000000
    default_timeout = 1000
    model_baudrate_table = {model: [1000000] for model in MODEL_BAUDRATE_TABLE}
    model_ctrl_table = {model: {} for model in MODEL_NUMBER_TABLE}
    model_encoding_table = MODEL_ENCODING_TABLE
    model_number_table = MODEL_NUMBER_TABLE
    model_resolution_table = MODEL_RESOLUTION_TABLE
    normalized_data = ["Present_Position", "Goal_Position"]
```

추가로 현재 `lerobot_robot_piper/motors/piper_motors_bus.py:109` 이후에 `MotorsBus`가 요구하는 serial-protocol method stub을 추가했습니다.

```python
def _assert_protocol_is_compatible(self, instruction_name: str) -> None:
    pass

def _handshake(self) -> None:
    pass

def _find_single_motor(self, motor: str, initial_baudrate: int | None = None) -> tuple[int, int]:
    raise NotImplementedError("Piper CAN bus does not support single motor discovery.")
```

의미:

- `PiperMotorsBus`가 LeRobot 0.4에서 추상 클래스 오류 없이 생성됩니다.
- CAN 기반 Piper에는 맞지 않는 serial-protocol 기능은 실제 구현하지 않고 compatibility layer로만 둡니다.
- 이 변경은 기능 추가보다 LeRobot 버전 차이로 생기는 실행 불가 문제를 막기 위한 호환성 패치입니다.

## 3. Follower 강제 parking 비활성화

WEGO 원본은 follower 연결 시 `connect(calibrate=True)` 흐름에서 항상 parking pose로 이동합니다. 로컬 수정본은 `park_on_connect` 설정을 추가하고 기본값을 `False`로 두었습니다.

Before: WEGO 원본 `lerobot_robot_piper/piper_follower.py:90`

```python
def connect(self, calibrate: bool = True) -> None:
    self.bus.connect()
    logger.info(f"{self} connected.")
    self.bus.enable_torque()
    logger.info(f"{self} torque on.")
    if calibrate:
        logger.info(f"{self} go to origin.")
        self.bus.parking()
```

After: 현재 `lerobot_robot_piper/piper_follower.py:92`

```python
def connect(self, calibrate: bool = True) -> None:
    self.bus.connect()
    logger.info(f"{self} connected.")
    self.bus.enable_torque()
    logger.info(f"{self} torque on.")

    if calibrate and self.config.park_on_connect:
        logger.info(f"{self} go to origin.")
        self.bus.parking()
```

의미:

- 프로그램을 재실행할 때 follower가 매번 초기 자세로 강제 이동하지 않습니다.
- leader/follower를 이미 맞춰둔 상태에서 teleoperation을 다시 시작해도 현재 자세를 유지합니다.
- 필요한 경우 `park_on_connect=true`로 원본과 유사한 동작을 다시 켤 수 있습니다.

## 4. Leader/Follower action offset 추가

WEGO 원본은 leader action을 그대로 follower goal로 보냅니다. 로컬 수정본은 첫 leader action과 follower 현재 joint position의 차이를 offset으로 계산하고, 이후 action에 더해 follower 현재 자세 기준의 상대 추종으로 변환합니다.

Before: WEGO 원본 `lerobot_robot_piper/piper_follower.py:148`

```python
def send_action(self, action: dict[str, Any]) -> dict[str, Any]:
    ...
    goal_pos = {}
    for key, val in action.items():
        if key.endswith(".pos"):
            goal_pos[key.removesuffix(".pos")] = val
        else:
            goal_pos[key] = val

    if self.config.max_relative_target is not None:
        present_pos = self.bus.sync_read("Present_Position")
        goal_present_pos = {key: (g_pos, present_pos[key]) for key, g_pos in goal_pos.items()}
        goal_pos = ensure_safe_goal_position(goal_present_pos, self.config.max_relative_target)
```

After: 현재 `lerobot_robot_piper/piper_follower.py:168`

```python
if self.config.use_action_offset:
    present_pos = self.bus.sync_read("Present_Position")
    if self._action_offset is None:
        if self.config.use_manual_action_offset:
            self._action_offset = self._manual_action_offset(goal_pos)
        else:
            self._action_offset = {key: present_pos[key] - val for key, val in goal_pos.items()}
        self._report_action_offset(goal_pos, present_pos, self._action_offset)

    goal_pos = {key: val + self._action_offset.get(key, 0.0) for key, val in goal_pos.items()}
```

```python
action_offset_joint2: float = 0.0
action_offset_joint3: float = 0.0
action_offset_joint4: float = 0.0
action_offset_joint5: float = 0.0
action_offset_joint6: float = 0.0
```

의미:

- leader/follower 시작 자세 차이를 자동으로 흡수합니다.
- 프로그램 재시작 후에도 follower가 현재 자세 기준으로 이어서 움직이기 쉽습니다.
- offset report를 통해 시작 자세 차이가 큰 joint를 확인할 수 있습니다.
- 반복 실험에서는 `ACTION_OFFSET_*` 값으로 수동 offset을 고정할 수 있습니다.

## 5. RealSense 동시 사용 안정화

WEGO 원본은 camera별 기본 `connect()`를 호출합니다. 로컬 수정본은 warmup 여부와 post-connect wait를 설정으로 제어합니다.

Before: WEGO 원본 `lerobot_robot_piper/piper_follower.py:99`

```python
for cam in self.cameras.values():
    cam.connect()
```

After: 현재 `lerobot_robot_piper/piper_follower.py:103`

```python
for cam in self.cameras.values():
    cam.connect(warmup=self.config.camera_connect_warmup)

if self.cameras and not self.config.camera_connect_warmup:
    time.sleep(self.config.camera_post_connect_wait_s)
```

의미:

- RealSense 두 대를 동시에 사용할 때 stream 시작 순서와 warmup으로 인한 timeout 문제를 줄입니다.
- `camera_connect_warmup`, `camera_post_connect_wait_s`, `realsense_warmup_s`로 실험 환경에 맞춰 조정할 수 있습니다.
- top/wrist 두 camera 구성을 dataset recording에 더 안정적으로 붙일 수 있습니다.
